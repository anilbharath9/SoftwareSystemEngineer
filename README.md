## 📚 Month-by-Month Study Guide

### 🗓️ Month 1

#### 🛠️ Book 1: Software Craftsmanship
* **Title:** *The Pragmatic Programmer (20th Anniversary Edition)*
* **Author:** Andrew Hunt & David Thomas
* **Focus:** It will teach you the core software development process.
* **Key Lessons:**
  * 🤖 **Automate repetitive tasks:** Save time and minimize errors.
  * 🕸️ **Think in systems:** See the bigger picture and understand how components interact.
  * 📈 **Continuously refine your skills:** Never stop learning and upgrading your toolkit.
* **Link:** [View PDF Document](Top%2012%20%20System%20Design%20Books/The%20Pragmatic%20Programmer,%2020th%20Anniversary%20Edition.pdf)

#### 🌐 Book 2: Distributed Systems
* **Title:** *Designing Data-Intensive Applications*
* **Author:** Martin Kleppmann
* **Focus:** It will teach you distributed systems concepts, trade-offs, and storage architectures.
* **Key Lessons:**
  * ⚖️ **Design for scalability:** Learn how to scale systems horizontally and vertically.
  * 🔄 **Understand tradeoffs:** Compare SQL vs NoSQL, replication strategies, and consistency models.
  * 🛡️ **Build fault-tolerant systems:** Ensure high availability even during partial system failures.
* **Link:** [View PDF Document](Top%2012%20%20System%20Design%20Books/Designing_Data_Intensive_Applications.pdf)

---

### 🗓️ Month 2

#### ✨ Book 3: Code Quality
* **Title:** *Clean Code: A Handbook of Agile Software Craftsmanship*
* **Author:** Robert C. Martin ("Uncle Bob")
* **Focus:** It will teach you practices to write clean, maintainable, and readable code.
* **Key Lessons:**
  * 🏷️ **Use meaningful names:** Intention-revealing variables, classes, and methods.
  * 🧩 **Keep functions small:** Do one thing and do it well.
  * 🧹 **Refactor aggressively:** Clean up code continuously as you build.
* **Link:** [View PDF Document](Top%2012%20%20System%20Design%20Books/Clean%20Code.pdf)

#### 📊 Book 4: Project Management
* **Title:** *The Mythical Man-Month: Essays on Software Engineering*
* **Author:** Frederick P. Brooks Jr.
* **Focus:** It will give you invaluable advice on managing and scheduling large-scale software projects.
* **Key Lessons:**
  * 👥 **Reduce coordination overhead:** Adding manpower to a late software project makes it later.
  * 📢 **Plan for communication costs:** Minimize complexity in team dynamics.
  * ⏱️ **Estimate realistically:** Understand the pitfalls of optimistic scheduling.
* **Link:** [View PDF Document](Top%2012%20%20System%20Design%20Books/The.Mythical.Man-Month.Essays.on.Software.Engineering.20th.Anniversary.Edition.pdf)

---

### 🗓️ Month 3

#### 🔄 Book 5: Refactoring
* **Title:** *Refactoring: Improving the Design of Existing Code*
* **Author:** Martin Fowler
* **Focus:** It will give you techniques to restructure code to enhance its maintainability without changing external behavior.
* **Key Lessons:**
  * 👃 **Remove code smells:** Identify duplication, long parameter lists, and large classes.
  * 🔀 **Simplify complex logic:** Refactor conditionals and loops for clarity.
  * 🪜 **Refactor incrementally:** Make small, safe steps supported by robust testing.
* **Link:** [View PDF Document](Top%2012%20%20System%20Design%20Books/Refactoring-Improving-the-Design-of-Existing-Code.pdf)

#### 🏚️ Book 6: Legacy Systems
* **Title:** *Working Effectively with Legacy Code*
* **Author:** Michael C. Feathers
* **Focus:** It will teach you techniques to understand and safely modify legacy (untested) systems.
* **Key Lessons:**
  * 🧪 **Write characterization tests:** Guard existing behavior before changing it.
  * 🔌 **Break dependencies safely:** Decouple classes to make testing easier.
  * 🐾 **Refactor in small steps:** Avoid massive rewrites that break production.
* **Link:** [View PDF Document](Top%2012%20%20System%20Design%20Books/Working%20Effectively.%20with%20Legacy%20Code.pdf)

---

### 🗓️ Month 4

#### 🏛️ Book 7: Software Architecture
* **Title:** *Software Architecture: The Hard Parts*
* **Author:** Mark Richards & Neal Ford
* **Focus:** It will teach you how to make better architectural decisions, analyze tradeoffs, and handle distributed architectures.
* **Key Lessons:**
  * ⚖️ **Analyze architectural tradeoffs:** There are no right answers in architecture, only trade-offs.
  * 🧱 **Optimize for constraints:** Balance decoupling, database replication, and workflow orchestration.
  * 🧗 **Balance scalability and complexity:** Choose the right service granularity.
* **Link:** [View PDF Document](Top%2012%20%20System%20Design%20Books/Software_Architecture_The_Hard_Parts_Neal_Ford.pdf)

#### 🗄️ Book 8: Database Internals
* **Title:** *Database Internals: A Deep Dive into How Distributed Data Systems Work*
* **Author:** Alex Petrov
* **Focus:** It will teach you how database storage engines and distributed database nodes operate under the hood.
* **Key Lessons:**
  * 🌲 **Learn B-trees and LSMs:** Master the fundamental data structures used by storage engines.
  * 👥 **Understand replication models:** Compare leader-follower, leaderless, and consensus-driven structures.
  * ⚡ **Optimize data access patterns:** Tailor queries based on hardware and read/write ratios.
* **Link:** [View PDF Document](Top%2012%20%20System%20Design%20Books/Database%20Internals.pdf)

---

### 🗓️ Month 5

#### 🚀 Book 9: Engineering Leadership
* **Title:** *Staff Engineer: Leadership Beyond the Management Track*
* **Author:** Will Larson
* **Focus:** It will give you techniques to grow as a technical leader without transitioning into people management.
* **Key Lessons:**
  * 🧭 **Lead through alignment:** Influence direction and align team goals without formal authority.
  * 🤝 **Drive cross-team initiatives:** Sponsor technical designs and champion architectural standards.
  * 🤝 **Build organizational trust:** Foster transparent communication and support peer growth.
* **Link:** [View PDF Document](Top%2012%20%20System%20Design%20Books/Staff%20Engineering.pdf)

#### 👥 Book 10: Team Leadership
* **Title:** *Extreme Ownership: How U.S. Navy SEALs Lead and Win*
* **Author:** Jocko Willink & Leif Babin
* **Focus:** It will teach you how to lead, build high-performance teams, and build a culture of accountability.
* **Key Lessons:**
  * 🛑 **Take full responsibility:** Own all outcomes—good and bad—under your purview.
  * 🗣️ **Communicate clearly:** Keep directives simple, concise, and understandable.
  * 🎯 **Prioritize and execute:** Remain calm under pressure, determine the highest priority, and act.
* **Link:** [View PDF Document](Top%2012%20%20System%20Design%20Books/Extreme%20Ownership.pdf)

---

### 🗓️ Month 6

#### 🎨 Book 11: Software Design
* **Title:** *A Philosophy of Software Design (2nd Edition)*
* **Author:** John Ousterhout
* **Focus:** It will teach you how to design deep modules and write simple, clean, and maintainable software.
* **Key Lessons:**
  * 🧠 **Reduce cognitive load:** Keep code simple to minimize the effort required to understand it.
  * 🙈 **Hide implementation details:** Design clean interfaces that abstract away complexity.
  * 🐋 **Design deep modules:** Create modules that provide powerful functionality with simple interfaces.
* **Link:** [View PDF Document](Top%2012%20%20System%20Design%20Books/A-philosophy-of-software-design-2nd-edition.pdf)

#### 🔍 Book 12: Debugging
* **Title:** *Why Programs Fail: A Guide to Systematic Debugging*
* **Author:** Andreas Zeller
* **Focus:** It will teach you scientific, systematic debugging procedures to diagnose and fix bugs.
* **Key Lessons:**
  * 🔬 **Form hypotheses methodically:** Use the scientific method to narrow down causes.
  * 🔁 **Reproduce bugs consistently:** Set up minimal, reproducible examples.
  * 🎯 **Isolate root causes:** Inspect state changes and traceback to find where execution went wrong.
* **Link:** [View PDF Document](Top%2012%20%20System%20Design%20Books/Why_Programs_Fail_Second_Edition_A_Guide_to_Systematic_Debugging.pdf)

## 🌟 Supplementary Resource

#### 🛡️ Book 13: Production & Reliability
* **Title:** *Release It!: Design and Deploy Production-Ready Software*
* **Author:** Michael T. Nygard
* **Focus:** Learn how to design systems that handle real-world deployment failures, scale gracefully, and remain highly resilient.
* **Key Lessons:**
  * 🔌 **Implement stability patterns:** Use Circuit Breakers, Bulkheads, and Timeouts to prevent cascading failures.
  * 📊 **Monitor SLA and metrics:** Track user experience, application health, and capacity.
  * 🏢 **Design for operations:** Make deployment, rollbacks, and configuration simple and predictable.
* **Link:** [View PDF Document](Top%2012%20%20System%20Design%20Books/Release%20It!_%20Design%20and%20Deploy%20Production-Ready%20Software.pdf)
